If configured, Jenkins will send out an notification to the specified recipients when a certain important event occurs.
  1. Every failed build triggers a new notification.
  2. A successful build after a failed (or unstable) build triggers a notification, indicating that a crisis is over.
  3. An unstable build after a successful build triggers a new notification, indicating that there's a regression.
  4. Unless configured, every unstable build triggers a new notification, indicating that regression is still there.
